Features
From description to device
⚡

MHDL Standard

One YAML/JSON file describes your entire hardware project. Git-friendly diffs, team-friendly workflows, AI-friendly structure.

✔

Design Rule Checks

Pin conflicts, I2C collisions, power budget, connection integrity, enclosure fit, mounting alignment — validated on every build.

⚙

Pluggable Backends

Wokwi, Arduino, OpenSCAD, KiCad by default. Swap in MicroPython, CadQuery, FreeCAD, or EasyEDA. Your choice.

○

3D Print Ready

Auto-generated parametric enclosures with snap-fit clips, screw mounts, cutouts for USB/LEDs/buttons, ventilation, and embossed labels.

⚗

MCP Native

Works with Claude, GPT, Gemini, and any MCP-compatible AI client. Hardware design as a conversation.

⧉

Full Pipeline

Circuit simulation, firmware generation, PCB export, enclosure design, BOM sourcing, and documentation — one command.

MCP Tools
Four tools, infinite hardware
meshforge-describe
Natural language to MHDL spec. Describe your board, get a buildable hardware definition.
meshforge-build
MHDL to all artifacts. Circuit, firmware, enclosure, BOM, and docs in one call.
meshforge-validate
6 Design Rule Checks. Nothing ships without passing validation.
meshforge-iterate
Patch and rebuild. Change the enclosure, swap the MCU, add components — instant rebuild.
Flagship Project
MeshCue Node

Built entirely with Forge

A dedicated mesh networking node — always-on, 3D printed, running on ESP32-S3. Relays signals, participates in consensus, displays live mesh status. Every artifact was generated from a single MHDL spec.
